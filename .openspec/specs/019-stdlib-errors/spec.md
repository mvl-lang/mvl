# Spec 019: Stdlib Structured Error Enums

**Status:** Accepted
**Issue:** #782
**ADR:** ADR-0032
**Phase:** 8

---

## Overview

All stdlib functions that can fail must return `Result[T, XxxError]` where `XxxError` is a domain-specific enum — not `Result[T, String]`. This enables exhaustive pattern matching (Req 3) and makes error kinds visible in the type system (Req 5).

---

## Requirement 1: Domain error enums defined [MUST]

Each stdlib module with fallible operations SHALL define a public error enum. The enum MUST include an `Other(String)` catch-all variant for unclassified errors.

**Implementation:**
- `std/net.mvl::NetError`
- `std/io.mvl::IoError`
- `std/regex.mvl::RegexError`
- `std/json.mvl::JsonError`
- `std/process.mvl::ProcessError`

#### Scenario: NetError exhaustive match

- GIVEN a program imports `std.net` and calls `tcp_connect`
- WHEN the call returns `Err(e)`
- THEN the compiler SHALL reject a match that does not cover all `NetError` variants (or use a wildcard)

**Tests:** `tests/stdlib/net_test.mvl::net_connect_refused_returns_err`

---

## Requirement 2: Result signatures updated [MUST]

All `pub builtin fn` declarations returning `Result[T, String]` SHALL be updated to `Result[T, XxxError]` where `XxxError` is the domain error for that module.

**Implementation:**
- `std/net.mvl` — all 6 fallible functions
- `std/io.mvl` — all 16 fallible functions
- `std/regex.mvl` — `compile`
- `std/json.mvl` — `decode`
- `std/process.mvl` — `spawn`, `stdin_write`, `stdout_read`, `stderr_read`, `wait`, `kill`

#### Scenario: Compile-time rejection of String error

- GIVEN a function `foo() -> Result[Int, String]`
- WHEN `foo` calls `tcp_connect` and tries to propagate the error with `?`
- THEN the compiler SHALL reject the propagation (type mismatch: `NetError` vs `String`)

---

## Requirement 3: Rust runtime mirrors error enums [MUST]

Each error enum defined in `std/xxx.mvl` SHALL have a corresponding Rust enum in `runtime/rust/src/stdlib/xxx.rs` with a "Mirrors the `XxxError` enum declared in `std/xxx.mvl`" comment. Variant names and ordering MUST match exactly.

**Implementation:**
- `runtime/rust/src/stdlib/net.rs::NetError`
- `runtime/rust/src/stdlib/io.rs::IoError`
- `runtime/rust/src/stdlib/regex.rs::RegexError`
- `runtime/rust/src/stdlib/process.rs::ProcessError`

#### Scenario: Mirror variant ordering

- GIVEN `NetError` has variants: ConnectionRefused(0), ConnectionReset(1), Timeout(2), AddressInUse(3), HostUnreachable(4), Other(5)
- WHEN the Rust runtime is compiled
- THEN the Rust `NetError` enum MUST have the same variant order as the MVL definition (discriminant values are determined by declaration order)

---

## Requirement 4: LLVM ABI supports enum errors [MUST]

The LLVM runtime SHALL represent error enum values using `LlvmEnumError { disc: u8, payload: [u8; 8] }`, matching the LLVM IR layout `{ i8, [8 x i8] }` generated by the codegen for payload enums.

**Implementation:** `runtime/llvm/src/abi.rs::LlvmEnumError`

Applies to: `std.net`, `std.io`, `std.regex` (LlvmResult path).

#### Scenario: LLVM enum error layout matches codegen

- GIVEN `LlvmEnumError` is `#[repr(C)] struct { disc: u8, payload: [u8; 8] }`
- WHEN the LLVM codegen generates GEP on the error value as `{ i8, [8 x i8] }`
- THEN field offsets SHALL match: discriminant at byte 0, payload at byte 1

---

## Requirement 5: Process LLVM gap acknowledged [SHOULD]

`std.process` LLVM functions use a different ABI (`MvlResult`) from the `LlvmResult` path used by net/io/regex. Full LLVM enum error support for `std.process` is deferred.

**Implementation:** `runtime/llvm/src/stdlib/process.rs` — retains string errors; TODO comment references #782.

---

## Requirement 6: Tests updated to match on variants [MUST]

Existing tests that compare error strings (`assert_eq(e, "connection refused")`) SHALL be updated to pattern-match on enum variants (`NetError::ConnectionRefused`).

**Implementation:** `tests/stdlib/net_test.mvl`, `tests/stdlib/process_test.mvl`, `tests/stdlib/regex_test.mvl`

#### Scenario: Error variant test

- GIVEN `tcp_connect("127.0.0.1", closed_port)` returns `Err(e)`
- WHEN the test matches on `e`
- THEN `NetError::ConnectionRefused` SHALL be the matching arm

**Tests:** `tests/stdlib/net_test.mvl::net_connect_refused_returns_err`
