# References

Full BibTeX available in [`references.bib`](https://github.com/LAB271/mvl_language/blob/main/docs/references.bib).

## Foundations

- **Curry (1934)** — First observation that types correspond to logical axiom-schemes
- **Howard (1969/1980)** — Formulae-as-types: propositions as types, proofs as programs
- **Hoare (1969)** — "An Axiomatic Basis for Computer Programming." Hoare triples {P} C {Q}
- **Martin-Lof (1972/1984)** — Intuitionistic type theory. Foundation for dependent types
- **Burstall, MacQueen, Sannella (1980)** — Hope language: algebraic data types, pattern matching
- **Girard (1987)** — Linear logic: each resource used exactly once
- **Meyer (1988)** — Design by Contract (Eiffel)
- **Plotkin & Pretnar (2009)** — Handlers of algebraic effects. ETAPS Test of Time Award 2022
- **Leijen (2014)** — Koka: programming with row-polymorphic effect types
- **Denning (1976)** — Lattice model of secure information flow

## Safety-Critical Standards

- **MISRA C:2023** — 143 rules + 16 directives restricting C for safety-critical systems
- **DO-178C (2011)** — Airborne software certification (avionics)
- **IEC 61508 (2010)** — Functional safety of programmable electronic systems

## Languages

- **CompCert** — Leroy et al., INRIA, 2006. Verified C compiler. ACM Software System Award 2021
- **Lean 4** — de Moura & Ullrich, 2021. Proof assistant + general programming. 9/11 on MVL scorecard
- **Idris 2** — Brady, 2021. Dependent types with quantitative type theory. Total by default
- **Pony** — Clebsch, 2015. Reference capabilities for compile-time data race freedom
- **Koka** — Leijen, 2014. Algebraic effects in a practical language
- **Lisette** — Ovejero, 2026. Rust-inspired language compiling to Go. [lisette.run](https://lisette.run/)

## LLM Research

- **MultiPL-E** — Cassano et al., IEEE TSE 2023. Cross-language LLM benchmark (18 languages)
- **Codex/HumanEval** — Chen et al., 2021. pass@k scales better for typed languages
- **DIN-SQL** — Pourreza & Rafiei, 2024. GPT-4 achieves 85%+ on Spider (DSL boundedness)
